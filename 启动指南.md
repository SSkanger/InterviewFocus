# 智能面试模拟系统 - 启动指南

## 问题诊断

您遇到的问题是"开始面试失败"，这通常是由于以下几个原因之一：

1. 后端服务器未正确启动
2. 后端初始化失败（摄像头、语音引擎等）
3. 前端无法连接到后端API
4. 系统依赖缺失

## 解决方案

### 方法1: 使用增强启动脚本（推荐）

1. 双击运行 `start-robust.bat` 文件
2. 等待脚本完成所有检查和启动步骤
3. 在浏览器中访问 http://127.0.0.1:5173

### 方法2: 手动启动（用于调试）

#### 步骤1: 启动后端服务器

1. 打开命令提示符（CMD）或PowerShell
2. 导航到项目目录：
   ```
   cd d:/PycharmProject/pywork/src
   ```
3. 安装Python依赖（如果尚未安装）：
   ```
   pip install -r requirements.txt
   ```
4. 启动后端服务器：
   ```
   python web_server.py
   ```
5. 等待看到"✅ 服务器准备就绪"消息

#### 步骤2: 测试后端API（可选）

1. 在另一个命令提示符窗口中，运行：
   ```
   python test_backend.py
   ```
2. 确保所有测试通过

#### 步骤3: 启动前端服务器

1. 打开新的命令提示符窗口
2. 导航到前端目录：
   ```
   cd d:/PycharmProject/pywork/app
   ```
3. 安装Node.js依赖（如果尚未安装）：
   ```
   npm install
   ```
4. 启动前端服务器：
   ```
   npm run dev
   ```
5. 等待看到"Local:"地址，通常是 http://127.0.0.1:5173

#### 步骤4: 访问系统

1. 在浏览器中打开 http://127.0.0.1:5173
2. 点击"开始面试"按钮

## 常见问题解决

### 问题1: 摄像头无法打开

1. 确保摄像头已连接并授权给应用程序
2. 检查是否有其他程序正在使用摄像头
3. 尝试重启计算机

### 问题2: 语音引擎初始化失败

1. 确保系统有音频输出设备
2. 尝试运行以下命令测试语音引擎：
   ```
   python -c "import pyttsx3; engine = pyttsx3.init(); engine.say('测试'); engine.runAndWait()"
   ```

### 问题3: 端口被占用

1. 检查端口5000和5173是否被其他程序占用
2. 使用以下命令查看端口占用情况：
   ```
   netstat -ano | findstr :5000
   netstat -ano | findstr :5173
   ```
3. 如果端口被占用，可以终止相关进程或修改配置文件中的端口号

### 问题4: 依赖安装失败

1. 确保Python版本为3.8或更高
2. 确保Node.js版本为16或更高
3. 尝试使用管理员权限运行命令提示符

## 系统要求

- Windows 10或更高版本
- Python 3.8+
- Node.js 16+
- 摄像头
- 麦克风（可选）
- 扬声器或耳机

## 技术支持

如果问题仍然存在，请检查：

1. 后端服务器窗口中的错误消息
2. 浏览器开发者工具中的网络请求错误
3. 确保防火墙没有阻止应用程序访问网络

您也可以尝试运行 `test_backend.py` 脚本来单独测试后端API是否正常工作。
